# Claude Cowork 五個排程提示詞

本文件提供五份「可直接複製使用」的 Claude Cowork 排程提示詞（Scheduled Task
prompt），對應每天五個分析時段：**03:00 / 07:00 / 12:00 / 17:00 / 22:00**
（Asia/Taipei）。

## 使用方式

1. 在 Claude（Cowork）中建立排程任務（Scheduled Task），共建立 5 個，分別對應
   下面 5 個時段。
2. 每個排程的 cron 建議設定為（皆為 **UTC**，Asia/Taipei = UTC+8，換算後如下）：

   | 時段（Asia/Taipei） | cron（UTC） |
   | --- | --- |
   | 03:00 | `0 19 * * *`（前一天 19:00 UTC） |
   | 07:00 | `0 23 * * *`（前一天 23:00 UTC） |
   | 12:00 | `0 4 * * *` |
   | 17:00 | `0 9 * * *` |
   | 22:00 | `0 14 * * *` |

3. 每個排程的 prompt 內容，直接複製下方對應區塊即可，不需要額外修改
   （已包含所有必要脈絡，因為每次排程觸發都是全新的 Session，不會記得先前對話）。
4. 請將下方 `g761007`、`daily-dispatch` 換成你實際使用的 GitHub 使用者名稱與
   repository 名稱（若與預設值相同則不需修改）。

## 共用規則（五個排程都適用）

- 一律使用 **Asia/Taipei** 日期與時間，不可直接使用執行環境的 UTC 日期。
- 只更新 `reports/YYYY-MM-DD.md`，使用文件中規定的固定 `<!-- slot: HH:MM:start -->`
  / `<!-- slot: HH:MM:end -->` 標記；同一時段重跑時「取代」該時段內容，不可重複
  附加、不可刪除或修改其他時段。
- 不可呼叫 Telegram、不可觸發或執行 `.github/workflows/publish-daily.yml`
  正式發布 workflow、不可讀取或寫入任何 GitHub Secrets。
- 完成後必須將變更 commit 並 push 到預設分支（`main`）。
- 全程使用台灣正體中文撰寫分析內容。

## 排程一：03:00 分析

```
你現在要執行 daily-dispatch 專案的每日新聞分析排程（第 1 / 5 個時段：03:00，Asia/Taipei）。

【專案位置】
- GitHub repository：https://github.com/g761007/daily-dispatch
- 若目前環境已連接使用者本機的 daily-dispatch 專案資料夾，直接在該資料夾工作。
- 若未連接本機資料夾，請在雲端工作區 git clone 上述 repository（分支：main）。

【第一步：確認日期與時段】
1. 取得目前 Asia/Taipei 的日期（YYYY-MM-DD）與時間，不可使用系統預設 UTC 日期。
2. 本次負責的時段固定為「03:00」。

【第二步：讀取或建立當日檔案】
1. 讀取 reports/YYYY-MM-DD.md（YYYY-MM-DD 為 Asia/Taipei 今天日期）。
2. 若檔案不存在，先建立檔案，內容需包含：
   - 檔案開頭 `# YYYY-MM-DD 每日分析`
   - `<!-- status: collecting -->`
   - 五個時段（03:00 / 07:00 / 12:00 / 17:00 / 22:00）各自的
     `<!-- slot: HH:MM:start -->` 與 `<!-- slot: HH:MM:end -->` 標記（可先留空，
     待各排程依序填入）。

【第三步：搜尋與分析】
1. 搜尋「自上一時段以來」的重要新消息（第一次執行時，搜尋近期重要新聞）。
2. 優先使用可信、可追溯來源；避免只轉貼標題；合併相同事件的重複報導。
3. 明確標註尚未證實的消息，不可把推測寫成已確認事實。
4. 若本時段沒有值得報告的新消息，內容必須寫：
   「本時段沒有發現足以改變今日判斷的重要新進展。」
   不可為了填滿內容而加入低品質新聞。

【第四步：寫入內容】
依照以下格式，寫入（或取代）reports/YYYY-MM-DD.md 中
`<!-- slot: 03:00:start -->` 到 `<!-- slot: 03:00:end -->` 之間的內容：

## 03:00 分析

### 重要更新

#### 事件標題
- 事件摘要：
- 最新進展：
- 重要性：
- 可能影響：
- 後續觀察：
- 來源：
  - [來源名稱](來源網址)

### 趨勢觀察
（本時段觀察到的共同趨勢）

### 低確定性消息
（尚未獲得足夠確認的內容；若無則寫「無」）

規則：
- 若該時段標記已存在，「取代」原有內容，不可重複附加。
- 不可修改或刪除其他時段（07:00 / 12:00 / 17:00 / 22:00）的內容。
- 保持 `<!-- status: collecting -->`（本排程不可將狀態改為 ready）。

【第五步：送出】
1. 確認變更只影響 reports/YYYY-MM-DD.md 的 03:00 區塊。
2. git add、commit（訊息例如：`content: 2026-07-22 03:00 分析`）、push 到 main。
3. 不呼叫 Telegram、不觸發 publish-daily workflow、不接觸任何 Secrets。
```
